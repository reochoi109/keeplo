# <type>(<scope>): <subject>
# 한 줄 요약 (현재 시제, 마침표 없이)
#
# 예시:
# feat(site): 사이트 등록 기능 추가
# fix(user): 로그인 실패 시 에러 메시지 수정
# chore: 패키지 의존성 업데이트

# --------------------
# 선택적으로 아래 내용을 작성해도 됩니다
#
# 본문 (무엇을, 왜 수정했는지 상세히)
#
# - 무엇을 변경했는가?
# - 왜 이 변경이 필요한가?
# - 부가적인 맥락이 있다면 설명
#
# Footer (선택)
#
# - 관련 이슈: #123
# - BREAKING CHANGE: 호환성 깨짐 등 주요 변경사항

# --------------------
# 커밋 타입 참고표

# | 타입         | 설명                                          |
# |--------------|-----------------------------------------------|
# | feat         | 새로운 기능 추가                               |
# | fix          | 버그 수정                                     |
# | docs         | 문서 수정 (README, 주석 등)                    |
# | style        | 코드 포맷 변경 (세미콜론 누락, 들여쓰기 등, 기능변경 없음) |
# | refactor     | 코드 리팩토링 (기능 변화 없음)                  |
# | perf         | 성능 향상 관련 수정                             |
# | test         | 테스트 코드 추가/수정                          |
# | build        | 빌드 시스템 수정 (예: webpack, package.json 등) |
# | ci           | CI 설정 수정 (예: GitHub Actions, Jenkins 등)  |
# | chore        | 기타 자잘한 작업 (패키지 업데이트, 설정파일 변경 등)     |
# | revert       | 이전 커밋 되돌리기                              |
